# How to Delegate Tasks to AI Agents (Actually)

In 2023 I spent four days trying to teach an AI agent to "manage my email."

Failed spectacularly. Know why? I was trying to delegate something vague instead of just starting with one specific task.

14 months later, I have 43 tasks running automatically every week. Haven't touched 37 of them since February.

## Start With Your Most Annoying Task

Every Monday at 9am I used to pull metrics from three places, make charts, post to Slack. 45 minutes. Every week. Same exact process.

Now? Agent does it at 8:55am while I'm still in bed.

That's the secret. Start with something boring. Something you hate. Something that never changes.

My current automated tasks:
- **Monday 9am**: Weekly metrics report (11 minutes saved)
- **Daily 3pm**: Expense receipt reconciliation (8 minutes saved)
- **Hourly**: New lead qualification and routing (5 minutes per lead)
- **Daily 8am**: Competitor monitoring (20 minutes saved)

These aren't complex AI breakthroughs. They're boring tasks with clear steps.

## Tasks That Actually Work for Delegation

**Perfect for agents:**
- Data reports on schedule
- Email sorting and filtering
- Form processing and routing
- Document generation from templates
- Meeting prep from calendar events

**Terrible for agents:**
- "Make my business better"
- Creative writing in your voice
- Tasks where requirements change weekly
- Anything requiring human judgment

I tried having agents write blog posts. Garbage. No personality, no edge, no humanity. Some things need brains.

## Writing Instructions That Don't Suck

My first instruction: "Send weekly metrics."

Useless.

My eighth version:
```
Every Monday at 9:00am EST:
1. Pull from Google Analytics: Pageviews, Users, Session Duration
2. Pull from Stripe: Revenue, MRR, Churn
3. Pull from HubSpot: New Leads count
4. Compare to same day previous week
5. Calculate percentage change
6. Format: Three bullets, max 15 words each
7. Post to #team-updates in Slack
8. Only mention changes under 5% if they reverse a trend
```

See the difference? Specific URLs. Exact metrics. Clear format. No ambiguity.

If you can't write instructions this specific, your task isn't ready to delegate.

## The First Month Reality

**Week 1**: You'll spend 3x longer setting it up than just doing it manually. My "15-minute setup" took 3 hours.

**Week 2**: You'll check every output manually. Despite it working fine. Trust comes slow.

**Week 3**: Something breaks. You fix it. Consider giving up. Don't.

**Week 4**: You forget it exists. Until someone asks about the report and you realize the agent sent it hours ago.

That's when you know it's working.

## Tools and Access (The Painful Part)

Your agent needs:
- **Data sources**: Read access to Google Analytics, Stripe, databases
- **Output destinations**: Send access to email, Slack, documents
- **Storage**: If needed, Dropbox or Google Drive access

Start with read-only everything. I ran my invoice agent for 8 weeks in "draft mode" before letting it actually send invoices. Paranoid? Maybe. But one wrong invoice is worse than 8 weeks of manual sending.

## Common Failures and Fixes

**"The output is inconsistent"**
Your instructions are vague. Add specific examples of good output.

**"It keeps breaking"**
You're trying to automate something with too many exceptions. Simplify the task.

**"It's not saving time"**
You're checking and editing every output. Either trust it or improve the instructions.

## The Setup That Actually Works

1. **Pick ONE task**. Not five. One.
2. **Do it manually one more time**, writing every single step
3. **Set up the agent** with those exact steps
4. **Run it supervised** for a week
5. **Let it run solo** once you trust it
6. **Add another task** only after the first is solid

Most people fail because they try to automate everything at once. Or they pick creative tasks. Or they write vague instructions.

Start boring. Start specific. Start small.
